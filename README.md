# Anchor

**Tagline:** A calmer way through your day.

Anchor is a daily support app for ADHD, autism, AuDHD, dyslexia, dyspraxia, anxiety-related executive dysfunction, sensory processing differences, burnout, chronic overwhelm, and anyone who benefits from external structure.

Anchor helps users understand what is happening, what matters next, how to regulate, and how to move through the day without relying only on memory, motivation, or willpower.

---

## Core Objective

Anchor turns the day into clear, manageable steps.

It helps users:

- Know what is next
- Reduce overwhelm
- Manage transitions
- Break down tasks
- Understand messages and emails
- Regulate sensory and emotional load
- Recover when plans change
- Communicate needs clearly
- Finish the day without shame

---

## Product Philosophy

Anchor should not behave like a productivity app built for neurotypical users.

It should not push:

- Streaks
- Hustle culture
- Rigid goals
- Shame-based accountability
- Productivity scoring
- “Try harder” language

Anchor is built around this principle:

> The user is not the problem. The environment, task structure, sensory load, ambiguity, and timing may be the problem.

Anchor helps modify those conditions.

---

## Core Promise

Anchor is a daily translation layer.

It translates:

- Long tasks into first steps
- Emails into key points
- Deadlines into reminders
- Appointments into preparation routines
- Overwhelm into reset options
- Routines into checklists
- Uncertainty into scripts
- Hard days into essentials
- Missed tasks into adjusted plans

The app’s central job is to reduce the amount of invisible cognitive work required to get through the day.

---

# Navigation Structure

Anchor uses consistent, predictable navigation.

## Main Sections

1. **Today**
2. **Now**
3. **Plan**
4. **TL;DR**
5. **Reset**
6. **Me**

Avoid hidden menus, surprise animations, sudden interface changes, and unexplained icons.

---

# Design Principles

## 1. Predictability

The same things should be in the same places every day.

Navigation should remain stable:

- Today
- Now
- Plan
- TL;DR
- Reset
- Me

---

## 2. One Decision at a Time

Each screen should ask for one thing.

### Avoid

> Plan your day, set goals, choose reminders, assign priorities, and review yesterday.

### Use

> What do you need help with right now?

Options:

- Know what’s next
- Start a task
- Understand a message
- Calm down
- Plan later
- Do essentials only

---

## 3. Regulation Before Productivity

If the user is overwhelmed, overstimulated, anxious, shutdown, tired, or confused, the app should not push action first.

Use:

> Do you need a reset before the next step?

---

## 4. Externalize Executive Function

Anchor should hold:

- Sequences
- Reminders
- First steps
- Deadlines
- Transition warnings
- Task context
- Routines
- Scripts
- Saved decisions
- Sensory needs

The user should not have to keep all of this in working memory.

---

## 5. No Shame States

Avoid:

- Failed
- Missed again
- You broke your streak
- Try harder
- You are behind

Use:

- This did not happen yet.
- Would you like to move, reduce, or delete it?
- The plan can change.
- Restart from any step.
- This is information, not failure.

# Task Breakdown Engine

Every task should be converted into a visible sequence.

## Standard Structure

1. Prepare
2. Start
3. Continue
4. Finish
5. Recover

---

## Example: Reply to Email

- Read with TL;DR Assist

### Start

- Choose reply type
- Write one sentence

### Continue

- Add any needed detail
- Check deadline or question

- Send

### Recover

- Pause for 1 minute
- Return to Now

---

## Standard TL;DR Output

### Main Point

What this is mostly about.

### Key Details

The important information, stripped down.

### What You Need To Do

Clear action items.

### Dates and Deadlines

Any time-sensitive details.

### Tone

What the message seems to sound like, without guessing hidden intent.

### Suggested Reply

A short optional response.

### Save to Today

Turn actions into reminders, tasks, routines, or calendar anchors.

---

## Example Output

### Main Point

Your appointment has moved to Thursday at 2:30 PM.

### Key Details

- The location is the same.
- You need to arrive 10 minutes early.
- Bring your ID.
- The appointment may take 45 minutes.

### What You Need To Do

1. Confirm the new time.
2. Add the appointment to Today.
3. Put your ID near your bag.

### Date / Time

Thursday, 2:30 PM  
Arrive by 2:20 PM

### Suggested Reply

Thanks for letting me know. I confirm the new appointment time for Thursday at 2:30 PM.

### Save to Today Buttons

- Add appointment
- Add reminder to bring ID
- Add leaving-home routine
- Save summary
- Delete pasted text

## Tone Explainer

### Purpose

Help the user understand social or professional tone without over-interpreting.

### Output Format

#### Likely Tone

Formal and neutral.

#### What Is Clear

They need a reply by Friday.

#### What Is Not Clear

They do not say whether the meeting is optional.

#### Do Not Assume

There is not enough evidence to know if they are annoyed.

#### Safe Reply

Thanks. Could you confirm whether this meeting is required or optional?

# Communication Support

## Purpose

Help users respond clearly, set boundaries, ask for clarity, and communicate needs.

## Script Library

### Ask for Clarity

Thanks for the message. Could you please clarify what you need from me and by when?

### Running Late

I’m running late. I’m still coming, but I need more time. I’ll update you when I know my arrival time.

### Need More Processing Time

I’ve seen this. I need some time to process it before I reply properly.

### Overwhelmed

I’m overwhelmed and need a quieter moment before I can respond properly. I’m not ignoring you.

### Set Boundary

I’m not available for this today. I can look at it on [date].

### Say No

Thanks for asking. I’m not able to do this.

### Change of Plan

I can adapt, but I need a few minutes to understand the new plan.

### Social Recovery

I enjoyed seeing you. I need quiet time now, so I may not reply quickly.

Anchor offers:

- TL;DR
- Action items

- Reply builder
- Save to Today

# Brand Voice

## Voice Qualities

- Calm
- Literal
- Respectful
- Practical
- Non-infantilizing
- Non-clinical unless needed
- Non-shaming
- Direct
